Project Specifications

Detailed Requirements for Each Project Option

Option 1: Restaurant Menu Website

Create a website for a restaurant of your choice (real or fictional).

Page 1: Landing Page (index.html) - Build from Scratch

Requirements:

  • Hero section with restaurant name, tagline, and appealing imagery
  • Navigation bar linking to Menu and About pages
  • Welcome section with 2-3 paragraphs about the restaurant
  • Featured dishes preview showing 3 signature items with images and descriptions
  • Footer with hours, location, and contact information

CSS Requirements:

  • Custom color scheme using RGB or HSL colors
  • At least 2 Google Fonts (one for headings, one for body)
  • Hover effects on navigation links
  • Box model spacing for visual organization

Page 2: Menu Page (menu.html) - Complete the Starter Code

You will receive:

  • HTML structure with menu sections (Appetizers, Entrees, Desserts)
  • At least 6 menu items already coded
  • Basic CSS layout structure
  • Wireframe showing the finished design

You must add:

  • Colors and background colors for sections and menu items
  • Typography styling (fonts, sizes, line-height)
  • Box model refinement (padding, margins, borders)
  • Price alignment (right-aligned or positioned)
  • Hover effects on menu items
  • "Chef's Special" or "Recommended" badges on 2-3 items using pseudo-elements (::before or ::after)
  • Images for at least 3 menu items (styled with borders or spacing)

CSS Techniques Required:

  • Pseudo-elements for badges
  • Hover effects with pseudo-classes
  • Box model manipulation
  • Text alignment or positioning for prices
  • Consistent spacing throughout

Page 3: About Page (about.html) - Build from Wireframe

You will receive:

  • Wireframe image showing the page layout
  • Content requirements list

Required Elements:

  • Restaurant story section (3-4 paragraphs)
  • Meet the team section with at least 2 staff members
    • Each staff member has a photo and bio
    • Photos should be floated left or right with text wrapping around them
  • Contact information clearly displayed (address, phone, email)
  • Hours of operation
  • Consistent header/navigation and footer with other pages

CSS Techniques Required:

  • Float staff photos with text wrapping
  • Clear floats appropriately
  • Box model spacing
  • Consistent styling with Pages 1 and 2

Option 2: Art Gallery Showcase Website

Create a website showcasing an art gallery or exhibition (real or fictional).

Page 1: Landing Page (index.html) - Build from Scratch

Requirements:

  • Hero section with gallery name and artistic tagline
  • Navigation bar linking to Gallery and Artist pages
  • Featured artwork spotlight showing 1 large prominent piece
  • Gallery preview with thumbnail images of 3-4 artworks
  • Welcome message (2-3 paragraphs) about the gallery or exhibition theme
  • Footer with gallery information

CSS Requirements:

  • Dramatic or elegant color scheme appropriate for an art gallery
  • Custom fonts that feel artistic/elegant
  • Featured artwork styled with border, shadow, or frame effect
  • Hover effects on thumbnails
  • Box model spacing

Page 2: Gallery Page (gallery.html) - Complete the Starter Code

You will receive:

  • HTML structure with 8-10 artwork containers already coded
  • Basic float-based layout structure
  • Wireframe showing the finished grid design

You must add:

  • Colors and background colors for artwork containers
  • Typography styling for artwork titles and artist names
  • Box model refinement (padding, margins, borders) for consistent spacing
  • Hover effects on artwork containers (shadow, border color, scale effect, etc.)
  • Consistent sizing for images
  • Any additional decorative styling

Image Requirements:

  • Use actual artwork images from the internet (museum websites, Google Arts & Culture, WikiArt, artist portfolios)
  • Include title and artist name for each artwork
  • All images must have alt text
  • You may cite the artist in small text below images or in HTML comments

CSS Techniques Required:

  • Float-based layout with proper clearing
  • Hover effects with pseudo-classes
  • Box model mastery for grid spacing
  • Consistent borders or shadows on image containers

Page 3: Featured Artist Page (artist.html) - Build from Wireframe

You will receive:

  • Wireframe image showing the page layout
  • Content requirements list

Required Elements:

  • Artist biography section (3-4 paragraphs about a specific artist or art movement)
  • Artist portrait or representative image floated with text wrapping
  • Notable works section showing 3-4 specific pieces by this artist with descriptions
  • Timeline or key facts about the artist's career
  • Artist quote or statement styled distinctively (centered, larger text, or special formatting)
  • Consistent header/navigation and footer with other pages

CSS Techniques Required:

  • Float artist portrait with text wrapping
  • Pseudo-elements (::before) for quotation marks or decorative elements
  • Box model spacing
  • Positioning or special styling for quotes
  • Consistent styling with Pages 1 and 2

Option 3: Event Poster/Flyer Website

Create a website promoting an event such as a concert, conference, tech meetup, festival, or campus event (real or fictional).

Page 1: Landing Page (index.html) - Build from Scratch

Requirements:

  • Hero section with large event title, date, and compelling imagery
  • Navigation bar linking to Schedule and Registration pages
  • Event overview (2-3 paragraphs) explaining what the event is about
  • Key highlights section with 3-4 main attractions or reasons to attend
  • Countdown or urgency element ("Only 2 weeks away!" or "Limited spots!")
  • Call-to-action button linking to registration page
  • Footer with event organizer info

CSS Requirements:

  • Bold, attention-grabbing color scheme
  • Large, impactful typography for event title
  • Call-to-action button with multiple states (:hover, :active, :focus)
  • Box model control for spacing and visual hierarchy
  • Hero section with positioning techniques

Page 2: Schedule/Lineup Page (schedule.html) - Complete the Starter Code

You will receive:

  • HTML structure with 4-6 speaker/performer cards OR time-based schedule already coded
  • Basic CSS layout structure
  • Wireframe showing the finished design

You must add:

  • Colors and background colors for cards/schedule items
  • Typography styling
  • Box model refinement (padding, margins, borders)
  • Images styled with spacing and borders (if using speaker/performer cards)
  • Hover effects on cards or schedule items
  • Special styling for keynote speakers or featured sessions (using specificity)
  • Float images beside text (if using speaker cards)

Choose One Format:

  • Speaker/Performer Cards: Image, name, title/role, and bio for each person
  • Time-Based Schedule: Time slots with session titles, descriptions, and location

CSS Techniques Required:

  • Float images with text (for speaker cards) or organized schedule layout
  • Hover effects with pseudo-classes
  • Box model spacing for cards
  • Specificity practice (style keynote differently from regular sessions)
  • Clear floats to manage layout

Page 3: Registration/Tickets Page (registration.html) - Build from Wireframe

You will receive:

  • Wireframe image showing the page layout
  • Content requirements list

Required Elements:

  • Ticket options section showing 2-3 different ticket tiers (Early Bird, General Admission, VIP, etc.)
  • Each tier displayed as a card with:
    • Ticket name
    • Price (with old price struck through if on sale)
    • Features included (bulleted list with checkmarks)
    • "Select" or "Buy Now" button (non-functional, styled only)
  • Important information section (what to bring, parking, policies - 3-4 paragraphs)
  • FAQ section with 3-5 questions and answers
  • Contact information for questions
  • Consistent header/navigation and footer with other pages

CSS Techniques Required:

  • Two-column layout for ticket tiers using floats (or single column if only 2 tiers)
  • Pseudo-elements (::before) to create checkmarks on feature lists
  • Box model mastery for card sizing and spacing
  • "Featured" or "Best Value" tier highlighted with special styling
  • Price display with strike-through on old price
  • Buttons with hover and active states
  • Clear floats appropriately

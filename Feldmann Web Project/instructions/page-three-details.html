Page 3 Wireframe Specifications

Independent Build - Complete Guidelines

Overview

For Page 3, students will build the entire page from scratch based on wireframe descriptions and requirements. No starter code is provided. Students must create both the HTML structure and all CSS styling to match the wireframe specifications.

Project 1: Restaurant - About Page (about.html)

Page Layout Description

Header/Navigation Section

  • Consistent with other pages (same navigation bar, same styling)
  • Links to: Home | Menu | About

Main Content Area (max-width: 1000px, centered)

Section 1: Page Title

  • Large heading: "About [Restaurant Name]"
  • Centered, prominent typography
  • Spacing below before next section

Section 2: Restaurant Story

  • Content: 3-4 paragraphs telling the restaurant's story
    • Paragraph 1: How the restaurant was founded
    • Paragraph 2: The chef's background and philosophy
    • Paragraph 3: What makes the restaurant special
    • Paragraph 4 (optional): Awards, recognition, or future plans
  • Styling: Comfortable reading width, good line-height, clear typography
  • Spacing: Generous margins above and below

Section 3: Meet the Team

  • Section heading: "Meet Our Team" (styled distinctly)
  • Team Member 1:
    • Image (150-200px square or circle) floated LEFT
    • Name as heading
    • Title/role (e.g., "Head Chef", "Sous Chef")
    • Bio paragraph (3-4 sentences) wrapping around the image
    • Clear spacing after this team member before the next
  • Team Member 2:
    • Image (150-200px square or circle) floated RIGHT
    • Name as heading
    • Title/role
    • Bio paragraph (3-4 sentences) wrapping around the image
    • Clear spacing after
  • Layout pattern: Alternating float directions creates visual interest

Section 4: Visit Us

  • Section heading: "Visit Us"
  • Two-column or organized layout containing:
    • Contact Information: Address (with icon or styled distinctly), Phone number, Email address
    • Hours of Operation: Days and times formatted clearly, Could be styled as a list or in a box
  • Styling: Consider using background color, border, or box to make this information stand out

Footer Section

  • Consistent with other pages
  • Copyright information, social media links (optional)

Visual Design Requirements

  • Consistent color scheme with Pages 1 and 2
  • Same fonts and typography system
  • Team member images should have rounded corners or circular shape
  • Hover effects optional but encouraged
  • Good use of whitespace and breathing room
  • Clear visual hierarchy (headings larger than body text)

CSS Techniques to Demonstrate

  • Float images left and right with text wrapping
  • Clear floats properly to prevent layout collapse
  • Box model for consistent spacing
  • Typography hierarchy
  • Optional: Positioning for contact info icons or decorative elements

Project 2: Art Gallery - Featured Artist Page (artist.html)

Page Layout Description

Header/Navigation Section

  • Consistent with other pages
  • Links to: Home | Gallery | Featured Artist

Main Content Area (max-width: 1000px, centered)

Section 1: Page Title

  • Large heading: "Featured Artist: [Artist Name]"
  • Optional: Birth-death years or time period below name
  • Centered, prominent typography

Section 2: Artist Portrait and Biography

  • Artist portrait/photo: 250-300px width, floated LEFT
  • Styling on image: Border, shadow, or frame effect to match gallery aesthetic
  • Biography text (3-4 paragraphs) wrapping around the portrait:
    • Paragraph 1: Early life and training
    • Paragraph 2: Artistic style and influences
    • Paragraph 3: Major accomplishments and recognition
    • Paragraph 4: Legacy or later work
  • Clear float after biography before next section

Section 3: Artistic Style Quote

  • Pull quote or artist statement styled distinctively
  • Should stand out from regular text (larger, centered, different color, or in a box)
  • Could use pseudo-elements (::before and ::after) to add decorative quotation marks
  • Spacing above and below
Example quote format: "Art is not what you see, but what you make others see."

Section 4: Notable Works

  • Section heading: "Notable Works"
  • Grid of 3-4 artworks by this specific artist
  • Each artwork includes:
    • Image (300-400px wide)
    • Title below image
    • Year created
    • Brief description (2-3 sentences)
  • Layout: Could be floated in a 2-column arrangement, or stacked vertically
  • Styling: Consistent spacing, borders or shadows on images

Section 5: Timeline or Key Facts

  • Section heading: "Career Highlights" or "Timeline"
  • Content: 4-6 key dates/events in the artist's career
    • Example: "1881 - First major exhibition in Paris"
    • Example: "1889 - Created [famous work]"
  • Styling: Could be a styled list, timeline format, or organized in boxes
  • Use of color, icons (using pseudo-elements), or creative formatting

Footer Section

  • Consistent with other pages

Visual Design Requirements

  • Gallery aesthetic maintained (possibly dark background with light text)
  • Same fonts as other pages
  • Artist portrait should be prominent and well-styled
  • Pull quote should be visually distinctive
  • Notable works should have consistent presentation
  • Timeline/facts should be easy to scan

CSS Techniques to Demonstrate

  • Float portrait with text wrapping
  • Pseudo-elements for decorative quote marks
  • Box model for spacing around all sections
  • Possibly positioning for pull quote
  • Clear visual hierarchy
  • Hover effects on notable works (optional)

Project 3: Event - Registration/Tickets Page (registration.html)

Page Layout Description

Header/Navigation Section

  • Links to: Home | Schedule | Registration

Main Content Area (max-width: 1000px, centered)

Section 1: Page Title

  • Subheading: "Secure Your Spot at [Event Name]"

Layout:

  • If 2 tiers: Each card is ~45% width with margin between
  • If 3 tiers: Each card is ~30% width with margin between
  • Cards floated left to sit side-by-side
  • All cards same height (use padding to balance)

Featured/Recommended Ticket:

  • One tier should be highlighted as "Best Value" or "Most Popular"
  • Distinctive styling: different background color, border, or badge
  • Could use a ribbon/badge in corner using positioning

Section 3: Important Information

  • Section heading: "What You Need to Know"
  • Content: 3-4 paragraphs covering:
    • What to bring to the event
    • Parking or transportation information
    • COVID policies or health guidelines
    • Cancellation policy
  • Styling: Clear, readable paragraphs with good spacing

Section 4: FAQ

  • Section heading: "Frequently Asked Questions"
  • Format: 4-5 questions with answers
    • Could use pseudo-elements (::before) to add "Q:" before questions
    • Could style questions as bold or larger text
    • Could add hover effects that highlight questions
  • Example questions:
    • "Can I transfer my ticket to someone else?"
    • "What if I can't attend?"
    • "Are meals included?"
    • "Is there a group discount?"

Section 5: Contact Us

  • Section heading: "Still Have Questions?"
  • Contact information:
    • Email address for questions
    • Phone number (optional)
    • Note about response time
  • Styling: Could be in a colored box or styled distinctively

Footer Section

  • Consistent with other pages

Visual Design Requirements

  • Ticket cards should be visually appealing and clear
  • Featured ticket must stand out noticeably
  • Checkmarks on features should be clear and consistent
  • Buttons should have hover states
  • Price display should be prominent and clear
  • Old prices (if applicable) must be struck through
  • Good spacing between all sections

CSS Techniques to Demonstrate

  • Float-based card layout (2-3 columns)
  • Pseudo-elements for checkmarks and potentially Q/A markers
  • Box model mastery for card sizing and spacing
  • Specificity when styling the featured ticket differently
  • Text-decoration: line-through for old prices
  • Button styling with :hover and :active states
  • Clear floats to prevent layout issues
  • Positioning (optional) for badges or ribbons

General Requirements for All Page 3 Builds

Technical Requirements

  • Valid HTML5 and CSS3
  • Semantic HTML structure
  • External CSS stylesheet only
  • Consistent navigation with Pages 1 and 2
  • Consistent header and footer with Pages 1 and 2
  • All images must have alt text
  • Proper file organization (css folder, images folder)

CSS Requirements Must Include

  • Box model manipulation throughout
  • At least one float with proper clearing
  • Consistent color scheme (RGB or HSL)
  • Typography hierarchy (headings vs body text)
  • At least one pseudo-class (:hover)
  • At least one pseudo-element (::before or ::after)
  • Consistent spacing and alignment

Design Quality

  • Matches the wireframe layout described
  • Professional appearance
  • Good use of whitespace
  • Clear visual hierarchy
  • Readable typography
  • Consistent with Pages 1 and 2
